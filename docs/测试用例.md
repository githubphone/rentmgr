# 出租屋管理系统测试用例

## 测试环境

## 一、后端 API 测试

### 1. 出租屋管理 (RentalHouse)

| 用例编号 | 用例名称 | 请求方法 | 请求路径 | 测试步骤 | 预期结果 |
|---------|---------|---------|----------|----------|----------|
| BE-H001 | 获取出租屋列表 | GET | /rental/house/list | 调用列表接口 | 返回分页数据，包含code=200 |
| BE-H002 | 根据ID获取出租屋 | GET | /rental/house/{id} | 调用详情接口 | 返回出租屋详细信息 |
| BE-H003 | 新增出租屋 | POST | /rental/house | 提交正确的出租屋数据 | 返回影响行数>0 |
| BE-H004 | 修改出租屋 | PUT | /rental/house | 提交修改后的数据 | 返回影响行数>0 |
| BE-H005 | 删除出租屋 | DELETE | /rental/house/{ids} | 传入存在的ID | 返回影响行数>0 |
| BE-H006 | 查询附近出租屋 | GET | /rental/house/nearby | 传入经纬度和半径 | 返回范围内的出租屋列表 |
| BE-H007 | 获取出租屋地图标注 | GET | /rental/house/map | 调用地图接口 | 返回地图标注列表 |

## 三、综合等级计算测试

### 测试场景

根据 `RentalHouseInspection.calculateOverallLevel()` 方法逻辑：

| 消防等级 | 数据维护等级 | 治安等级 | 预期综合等级 |
|---------|-------------|---------|-------------|
| 1 (一级) | 1 (一级) | 1 (一级) | green (绿色) |
| 1 (一级) | 1 (一级) | 2 (二级) | yellow (黄色) |
| 1 (一级) | 2 (二级) | 2 (二级) | yellow (黄色) |
| 2 (二级) | 2 (二级) | 2 (二级) | red (红色) |
| 3 (三级) | 3 (三级) | 3 (三级) | red (红色) |

## 五、接口测试命令示例

### 使用 curl 测试后端 API

```bash
# 测试获取出租屋列表
curl -X GET "http://localhost:8080/rental/house/list" \
  -H "Content-Type: application/json"

# 测试新增出租屋
curl -X POST "http://localhost:8080/rental/house" \
  -H "Content-Type: application/json" \
  -d '{"houseName":"测试出租屋","address":"测试地址","latitude":22.123,"longitude":113.456}'

# 测试新增检查记录
curl -X POST "http://localhost:8080/rental/inspection" \
  -H "Content-Type: application/json" \
  -d '{"houseId":1,"inspectorId":1,"fireLevel":"1","dataLevel":"1","securityLevel":"1"}'
```
