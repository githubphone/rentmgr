## 二、角色权限分类体系

### 2.1 角色定义

系统角色分为以下几类：

| 角色 | 角色标识 | 数据权限范围 |
|-----|---------|-------------|
| 管理者 | landlord | 仅查看自己名下的出租屋 |
| 网格员 | grid_member | 查看网格内所有出租屋 |

### 2.2 数据权限实现

需要在现有权限体系中新增数据权限规则：

1. **管理者权限**：通过 `rental_manager_house` 表关联管理者与出租屋
2. **网格员权限**：通过 `rental_role_grid` 表关联角色与网格

```sql
-- 管理者出租屋关联表
CREATE TABLE rental_manager_house (
    id BIGSERIAL PRIMARY KEY,
    manager_id BIGINT NOT NULL COMMENT '管理者ID',
    house_id BIGINT NOT NULL COMMENT '出租屋ID',
    relation_type VARCHAR(20) DEFAULT 'owner' COMMENT '关系类型：owner=业主/trustee=托管人',
    create_by VARCHAR(64) DEFAULT '' COMMENT '创建者',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by VARCHAR(64) DEFAULT '' COMMENT '更新者',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '更新时间'
);

-- 角色网格关联表
CREATE TABLE rental_role_grid (
    id BIGSERIAL PRIMARY KEY,
    role_id BIGINT NOT NULL COMMENT '角色ID',
    grid_id BIGINT NOT NULL COMMENT '网格ID',
    create_by VARCHAR(64) DEFAULT '' COMMENT '创建者',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
);
```

### 2.3 涉及的代码文件

- **后端：** `ruoyi-system/src/main/java/com/ruoyi/system/service/impl/SysRoleServiceImpl.java`
- **数据权限注解：** 使用 `@DataScope` 注解扩展

## 七、分色等级体系

### 7.1 分色维度

系统对出租屋进行三维度评估：

| 评估维度 | 等级划分 | 说明 |
|---------|---------|------|
| 消防等级 | 一级/二级/三级 | 根据消防设施检查情况 |
| 数据维护等级 | 一级/二级/三级 | 根据信息完整度、更新频率 |
| 治安等级 | 一级/二级/三级 | 根据治安事件发生情况 |

### 7.2 综合评级规则

```
三项绿色  →  绿色（优秀）
两项绿色  →  黄色（良好）
其他情况  →  红色（需整改）
```

### 7.3 分色实现逻辑

```java
// 综合评级计算逻辑
public String calculateOverallLevel(FireLevel fire, DataLevel data, SecurityLevel security) {
    int greenCount = 0;
    if ("1".equals(fire)) greenCount++;
    if ("1".equals(data)) greenCount++;
    if ("1".equals(security)) greenCount++;

    if (greenCount >= 3) return "green";
    if (greenCount >= 2) return "yellow";
    return "red";
}
```

### 7.4 数据库设计

`rental_houses` 表新增字段：

```sql
ALTER TABLE rental_houses ADD COLUMN fire_level char(1) DEFAULT '3' COMMENT '消防等级：1-一级 2-二级 3-三级';
ALTER TABLE rental_houses ADD COLUMN data_level char(1) DEFAULT '3' COMMENT '数据维护等级：1-一级 2-二级 3-三级';
ALTER TABLE rental_houses ADD COLUMN security_level char(1) DEFAULT '3' COMMENT '治安等级：1-一级 2-二级 3-三级';
ALTER TABLE rental_houses ADD COLUMN overall_level char(1) DEFAULT 'red' COMMENT '综合等级：green-绿色 yellow-黄色 red-红色';
```

### 7.5 涉及的代码文件

- **实体类：** `RentHouse.java`
- **Service：** 分色计算逻辑
- **前端：** 地图标记颜色、列表页颜色标识

| 状态 | 状态码 | 说明 |

| 角色 | 操作权限 |

- 需要填写闭环说明
- 需要上传处理凭证（可选）
- 记录闭环时间和操作人
