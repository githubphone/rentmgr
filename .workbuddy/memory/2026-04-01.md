## 修复 mobile 端 Taro + Vue3 启动错误

**问题1：** `app.ts` 中 `app.config.compilerOptions = {}` 在 runtime-only 构建下无效，删除该代码。

**问题2（根本原因）：** `app.ts` 写法完全错误。Taro 的入口文件规范：
- `createApp()` 传配置对象（生命周期等），**不传 SFC 组件**
- 直接 `export default App`（app 实例），**不调用 `app.mount()`**
- Taro 自己负责挂载，手动 mount 会导致 `createVue3App` 收到错误对象

正确写法：
```ts
const App = createApp({ onShow() {...} })
App.use(pinia)
App.config.globalProperties.config = config
export default App
```

## 修复 app.scss CSS 路径错误

**错误：** `[postcss] ENOENT: no such file or directory, open '../scss/colorui.css'`

**原因：** `mobile/src/static/scss/index.scss` 中引用 `../scss/colorui.css`，但 `colorui.css` 就在同目录（`src/static/scss/`），路径写错了。

**修复：** 将 `@import "../scss/colorui.css"` 改为 `@import "./colorui.css"`。

## 修复 Taro 4.1.9 components-loader.js bug（二次修复）

**问题：**
`@tarojs/components` 的 `initVue3Components` 函数在 Taro H5 构建时，被插入到 Taro 生成的 entry 代码里，在 `createVue3App()` 调用之前执行，传入的第一个参数是 Vue 组件对象（非 app 实例），导致 `app.config` 为 undefined，触发 `Cannot set properties of undefined (setting 'isCustomElement')` 错误。

**根因：**
Taro 4.1.9 的 `plugin-platform-h5` 在生成的入口代码中：
```js
initVue3Components(component, list)   // component 是 SFC 组件对象，无 .config
var inst = createVue3App(component, h, config)  // 这里才创建 app 实例
```
`initVue3Components` 访问了不存在的 `component.config`。

**修复方案：**
使用 `pnpm patch` 对 `@tarojs/components@4.1.9` 的 `lib/vue3/components-loader.js` 打补丁，加入保护判断：
- patch 文件：`mobile/patches/@tarojs__components@4.1.9.patch`
- package.json 中添加 `pnpm.patchedDependencies` 确保重装后自动应用


**问题：**
- `app.ts` 中设置 `app.config.compilerOptions = {}` 导致 Vue runtime-only 构建警告
- `@tarojs/components` loader 报错 `Cannot set properties of undefined (setting 'isCustomElement')`

**原因：**
Taro 使用 runtime-only 构建，`app.config.compilerOptions` 在此模式下不可用。编译器选项（`isCustomElement`）应通过 `@vitejs/plugin-vue` 的 `compilerOptions` 传入，而非在运行时通过 `app.config` 设置。

**修复：**
删除 `mobile/src/app.ts` 中对 `app.config.compilerOptions` 的赋值代码（共3行）。
`config/index.ts` 的 `h5.vitePlugins` 中已正确配置 `isCustomElement: (tag) => tag.startsWith('taro-')`，无需额外修改。
